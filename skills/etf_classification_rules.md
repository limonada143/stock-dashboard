# ETF 분류 규칙 (자산운용사 무관)

자산운용사(Tiger, Kodex, Kindex, Hanaro, ACE 등)가 달라도 **추종 지수·테마가 같으면 동일 카테고리로 분류**한다.
종목명 매칭은 대소문자 무관, 한/영 혼용 허용.

---

## 분류 우선순위

규칙은 위에서 아래 순서로 적용한다. 먼저 매칭된 규칙이 우선된다.

---

## 규칙 목록

### 1. S&P500
**조건**: 종목명에 `S&P500`, `S&P 500`, `s&p500`, `sp500` 중 하나라도 포함
**카테고리**: `S&P500`
**예시**: TIGER S&P500, KODEX 미국S&P500, SOL 미국S&P500, TIGER미국S&P500

---

### 2. KOSDAQ (KOSPI보다 먼저 체크)
**조건**: 종목명에 `KOSDAQ`, `코스닥` 중 하나라도 포함
**카테고리**: `KOSDAQ`
**예시**: KODEX 코스닥150, KODEX 코스닥150레버리지, ACE 코스닥150, TIME 코스닥액티브

---

### 3. 나스닥
**조건**: 아래 중 하나라도 해당
- 종목명에 `나스닥`, `NASDAQ`, `Nasdaq` 포함
- 종목명에 `미국테크TOP`, `미국 테크 TOP` 포함 (미국 테크 상위 종목 ETF)
- 종목명에 `QQQ` 포함

**카테고리**: `나스닥`
**예시**: TIGER 나스닥100, TIGER미국나스닥100, TIME 미국나스닥100액티브, TIGER 미국테크TOP10 INDXX

---

### 4. KOSPI
**조건**: 아래 중 하나라도 해당
- 종목명에 `코스피`, `KOSPI` 포함
- 종목명이 `운용사 + 200/100/레버리지/인버스` 패턴이고, 외국 지수 명시 없음
  - 지원 운용사: KODEX, TIGER, ACE, KINDEX, HANARO, SOL, RISE, PLUS, 1Q

**카테고리**: `KOSPI`
**예시**: KODEX 200, KODEX 레버리지, KODEX 코스피, KODEX 코스피100, 1Q 200액티브

> **판단 기준**: KODEX 뒤에 별도 지수명(미국, 나스닥, S&P500, 반도체 등)이 없으면 KOSPI 추종으로 간주.
> `KODEX 코스닥150레버리지`는 KOSDAQ 규칙(#2)에 먼저 걸려 KOSPI 규칙 적용 안 됨.

## 적용 범위

- `index.html` 대시보드의 **Total / User / Husband** 탭 섹터 도넛 차트
- 구현 함수: `holdingToSector(name, cat)` — 이름 규칙 우선, 없으면 `catToSectorG(cat)` 폴백
- `portfolio.json`, `portfolio_husband.json`의 `category` 필드 입력 시에도 이 규칙을 따른다
