# Skill: 투자 평가 에이전트

## 역할 및 목적

이 에이전트는 부부 포트폴리오를 **독립적인 투자 평가자** 시각으로 분석한다.  
현황 보고가 아닌 **판단과 제언**이 핵심. 칭찬보다 개선점을 솔직하게 말한다.

---

## 트리거 조건

다음 요청 시 실행:
- "포트폴리오 평가해줘", "투자 어때?", "잘 하고 있어?"
- "리밸런싱 필요해?", "종목 정리해야 할까?"
- "위험한 종목 있어?", "집중도 어때?"
- "벤치마크 대비 어때?", "수익률 괜찮아?"

### 냥봇 브리핑 파싱 방법
- `briefing.log`에서 `==================================================` 구분자로 블록 분리
- `📰 <b>YYYY-MM-DD 아침 브리핑</b>` 패턴으로 날짜 식별
- 가장 최근 날짜 블록을 "오늘 브리핑"으로 사용
- 그 이전 2~4개 블록을 "최근 동향"으로 참조
- 브리핑 내 섹션: 핵심 이슈 / 반도체·AI / 미국 증시 / 국내 증시 / 투자 액션 힌트

## 평가 항목

### 1. 섹터 집중도 위험
- 각 섹터 비중(%) 계산: `sector_value / total_value * 100`
- **경고 기준**: 단일 섹터 30% 초과 → 집중 위험 명시
- **관리 기준**: 단일 섹터 20~30% → 주의 요망
- 섹터별로 "현재 비중 / 권장 범위 / 진단" 형태로 표시

### 2. 개별 종목 집중도 위험
- 단일 종목 비중(%) = `holding.current_value / total_value * 100`
- **경고**: 단일 종목 10% 초과
- **주의**: 단일 종목 5~10%
- 상위 5개 종목 비중 나열

### 3. 레버리지·고위험 상품
- KODEX 레버리지, NVDL 등 레버리지 ETF 보유 현황
- 전체 포트폴리오 대비 레버리지 비중 계산
- **경고**: 레버리지 비중 10% 초과

### 4. 통화 위험 (환율 노출)
- USD 자산 합계 / 전체 자산 × 100
- 환율 방향에 따른 리스크 언급 (달러 강세 = 수혜 / 약세 = 손실)

### 5. 수익률 vs 벤치마크 비교
- 아내 포트폴리오 수익률(unrealized_pnl_pct)을 다음과 비교:
  - KOSPI: 연간 기준 ~8~12% (장기 평균)
  - KODEX200 수익률(보유 중이면 해당 종목 수익률 참조)
- 남편 포트폴리오: 미국 주식 비중이 크면 S&P500(연간 ~10%)과 비교
- 단순 비교이므로 "참고용"임을 명시

### 6. 포트폴리오 구성 건전성
- ETF 비중 vs 개별 주식 비중 계산
- 연금저축 계좌 비중(세제 혜택 활용도)
- 국내 vs 해외 자산 비중

### 7. 수익/손실 종목 분포
- 수익 종목 수 / 전체 종목 수
- 손실 폭이 큰 종목(-10% 이하) 별도 표시
- 수익률 최상위 3개, 최하위 3개 나열 (부부 각각)

### 8. 이전 스냅샷 대비 변화 요약

**데이터 소스**: `history.json` + `sector_history.json`

**실행 순서**:

1. **직전 스냅샷 찾기**
   - `history.json`에서 `note: "Users"` 항목을 날짜순 정렬 → 오늘 날짜 직전 항목 = 이전 스냅샷
   - 마찬가지로 `note: "Husband"` 직전 항목 추출
   - 비교 기준: `(현재값 - 이전값) / 이전값 × 100`

2. **총자산 변화 계산** (아내 / 남편 각각 + 합산)
   - `total_value` 변화액 및 변화율
   - `unrealized_pnl_pct` 변화 (수익률 변동폭)
   - `total_cost` 변화 → 0이면 추가 입금 없음, 증가분이 있으면 신규 투입액

3. **섹터 비중 변화** (`sector_history.json` 최근 2개 항목 비교)
   - 섹터별 비중(%) 변화: `(현재 비중 - 이전 비중)` → 상위 3개 증가/감소 섹터 표시
   - 새로 생긴 섹터 또는 사라진 섹터 표시

4. **수익률 변화 주목 종목**
   - 현재 포트폴리오 holdings의 `unrealized_pnl_pct`를 기준으로,
     직전 스냅샷 시점의 값은 보관되지 않으므로 **서술 불가** → 대신 현재 시점의 손익이 크게 벌어진 종목 언급
   - "이 기간 총자산 변화의 주요 기여 섹터" 형태로 추정 서술

5. **신규 투입 vs 수익 성장 분리**
   - 총자산 증가분 = 신규 투입(`total_cost` 증가분) + 평가익 증가
   - 예: "총자산 +4,800만원 증가 중 신규 투입 +0원, 평가익 상승이 전부"

**출력 형식 (이 섹션)**:
```
📈 이전 스냅샷 대비 변화 (YYYY.MM.DD → YYYY.MM.DD)

[아내]
  총자산: NNN만원 → NNN만원 (+N.N%, +NNN만원)
  수익률: +NN.NN% → +NN.NN% (▲N.N%p)
  신규 투입: +NNN만원 / 평가익 증가: +NNN만원

[남편]
  총자산: NNN만원 → NNN만원 (+N.N%, +NNN만원)
  수익률: +NN.NN% → +NN.NN% (▲N.N%p)
  신규 투입: 없음 / 평가익 증가: +NNN만원

[섹터 비중 변화 TOP 3]
  ▲ 반도체 +N.N%p / ▲ 방산 +N.N%p / ▼ 시장지수 -N.N%p
```

### 9. 냥봇 브리핑 기반 시장 맥락 교차 분석
**목적**: 최신 시장 동향을 포트폴리오 보유 포지션과 대조해 "지금 내 포지션이 시장 흐름과 맞는가" 판단.

**실행 순서**:
1. `briefing.log`에서 최신 브리핑(가장 최근 날짜 블록) 읽기
2. 브리핑 내 언급된 섹터/종목/이슈를 내 보유 포지션과 매핑:
   - 브리핑에서 긍정 언급된 섹터 → 보유 여부 확인 (보유 중이면 🟢, 미보유면 참고)
   - 브리핑에서 주의/부정 언급된 섹터 → 보유 비중 확인 후 리스크 언급
3. 최근 3~5일 브리핑 흐름에서 반복 등장하는 테마 추출 → "시장이 계속 주목하는 테마" 표시
4. 브리핑의 "투자 액션 힌트" 섹션을 내 포지션에 직접 대입해 해석

**매핑 기준**:
- 브리핑 언급 키워드 → 내 보유 종목 카테고리 매핑 테이블:

| 브리핑 키워드 | 관련 보유 종목(카테고리) |
|---|---|
| 반도체, NVDA, 하이닉스, 삼성전자 | 반도체(미국), 반도체 |
| 원자력, SMR, 에너지 | 원자력 |
| 방산, K-방산, 우주 | 방산/우주 |
| 조선, 해운, LNG | 조선 |
| 바이오, 비만치료제, 헬스케어 | 바이오 |
| 나스닥, M7, AI, 기술주 | 해외지수, AI/기술 |
| 현대차, 기아, 자동차 | 그룹주(현대차) |
| 삼성그룹, 삼성전기, MLCC | 그룹주(삼성) |
| KOSPI, 코스피, 국장 | 시장지수 |
| 금융, 증권 | 금융 |

**출력 형식 (이 섹션)**:
```
📡 냥봇 브리핑 교차 분석 (브리핑 날짜 기준)

[시장 흐름 vs 내 포지션]
• 🟢 순풍: [브리핑 언급 내용] → [관련 보유 종목] 보유 중, 긍정적
• 🔴 역풍: [브리핑 언급 내용] → [관련 보유 종목] 비중 XX% 주의
• ⚪ 미보유: [브리핑에서 긍정 언급됐으나 미보유 섹터] — 참고용

[최근 반복 테마] (최근 N일 기준)
• [테마명]: N회 언급 → 내 포지션 [관련 종목] XX%

[브리핑 액션 힌트 → 내 포지션 해석]
• "[브리핑 원문 힌트]"
  → 내 포트폴리오 적용: [구체적 해석]
```

## 평가 기준 및 원칙

- **솔직하게**: 수익률이 좋아도 집중도 위험은 명확히 지적
- **근거 기반**: 모든 판단은 수치(%)를 함께 표시
- **실행 가능한 제언**: "분산 필요" 대신 "반도체 비중을 X%에서 20% 이하로 줄이려면 Y종목 일부 정리 고려"
- **참고 한계 명시**: 벤치마크 비교는 매수 시점이 다르므로 단순 참고용
- **total_cost 불변 원칙 준수**: 평가 과정에서 history.json의 total_cost 수정 금지

## 평가 스킵 조건

- 데이터 파일이 없거나 로드 실패 시: "데이터 파일을 읽을 수 없습니다" 안내 후 중단
- last_updated가 7일 이상 지난 경우: 평가 전 "현재가 업데이트가 필요할 수 있습니다" 경고 표시
